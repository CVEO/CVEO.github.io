# 内容管理规范

## 总览
- 站点采用 Astro + Tailwind 构建，内容以 Markdown 为主。
- 所有内容更新通过编辑 `src/content` 与 `src/pages` 完成。
- 新闻单一文件维护，论文采用内容集合，专利与软著使用 Markdown 表格。

## 新闻动态
- 文件位置：`src/content/news.md`
- 书写格式：
  - 每条新闻一行，使用二级标题：
  - `## YYYY-MM-DD 具体内容`
- 示例：
  - `## 2023-10-01 课题组最新成果发表于《遥感学报》`
- 自动排序与锚点：
  - 页面自动按日期降序排序
  - 每条新闻自动生成锚点 `#news-YYYY-MM-DD-内容摘要`

## 期刊论文
- 目录：`src/content/papers/`
- 每篇论文一个 Markdown 文件，使用 Frontmatter 结构化字段：
  - `title`: 标题
  - `journal`: 期刊
  - `date`: 发表日期 `YYYY-MM-DD`
  - `partition`: 分区标注，如 `【1区Top】`
  - `equalFirst`: 是否共同一作，布尔
  - `corresponding`: 通讯作者
  - `authors`: 作者数组
- 示例：
```
---
title: 遥感学报示例论文
journal: 遥感学报
date: 2023-10-01
partition: "【1区Top】"
equalFirst: true
corresponding: 张三
authors:
  - 张三
  - 李四
---
```

## 专利与软著
- 维护位置：`src/pages/research.astro` 中表格
- 建议迁移到 `src/content/patents.md` 并采用 Markdown 表格：
```
| 类别 | 名称 | 日期 |
| --- | --- | --- |
| 专利 | 示例名称 | 2023-08-01 |
| 软著 | 示例名称 | 2023-05-12 |
```

## 团队成员
- 集合目录：`src/content/members/`
- 字段：
  - `name`: 姓名
  - `role`: 角色或职称
  - `group`: `PhD | Master | Undergrad | Advisor`
  - `photo`: 照片路径或 URL
  - `bio`: 简介
- 照片规范：
  - 统一 600×800 像素
  - 浅色底照
  - 文件命名建议：`姓名-600x800.jpg`
- 教授与研究员独立页面：
  - 位置：`src/pages/team/`，示例 `pi.astro`

## 其他板块
- 采用 Markdown + 静态 HTML 混合方案
- 建议在 `src/pages` 下创建对应页面，并在 `src/content` 存放数据

## 英文版预留
- 建议目录结构：
  - `src/pages/en/`
  - `src/content/en/`
- 与中文站点字段一致
